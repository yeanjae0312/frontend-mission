#  주차별 기능 구현 :runner:
## 1주차
### :bulb: 개발환경 설정 & Warm up
<img src="https://user-images.githubusercontent.com/32696209/148894408-63531e69-79ee-4bfc-abdd-9df3afcc806b.png" width="600"/>

### :pushpin: 구현 기능
- input에 입력한 값이 실시간으로 반영되어 바로 밑에 표시되게 하기
- '회전' 버튼 클릭 시 input에 입력된 문자열과 바로 밑에 표시되는 문자열이 왼쪽으로 1칸씩 이동되게 하기
- '알림' 버튼 클릭 시 input에 입력된 문자열과 이 버튼을 누른 횟수를 보여주는 알림창 띄우기
### :zap: 접근 방법
- input에 입력한 값이 실시간으로 반영되는 기능이 v-model을 이용해 구현했었는데, CJK 언어권에서는 한글자씩 밀려서 반영되는 것을 알게 되었고 그래서 input속성을 사용해 적용시켰더니 정상적으로 동작하게 됨
- 모달 등을 구현할 때 한 페이지 내에서 기능을 구현하는 것이 아니라 최대한 컴포넌트로 나눠서 유지보수하기 좋은 쪽으로 코드를 짜려고 노력함
### :zap: 아쉬운 점
- TDD방법론으로 테스트 코드를 짜는데에 어려움을 겪었으나 추후 그룹별 코멘트와 강사님의 코멘트를 참고해서 작성 할 수 있었음

## 2주차
### :bulb: 쇼핑몰 상품 상세 정보 페이지 구현하기
<img src="https://user-images.githubusercontent.com/32696209/149610276-1a0405c4-706e-4507-b72c-2b7f74c1bd32.png" width="200" />

### :zap: 접근 방법
- 코드를 짜기 전에 기획을 하고 프로토타입을 제작함
- 로직을 구현하기 전에 기획하고 산출된 데이터를 미리 세팅함
- 제품이미지와 리뷰이미지의 비율을 어떤 길이의 이미지가 와도 1:1로 보여지게 하기 위한 스타일을 지정함
- 요구사항엔 없는 즐겨찾기, 찜하기 기능 구현함
- 할인에 대한 불린값을 받아와서 할인이 있을때와 없을때를 동적으로 처리함
- 기능과 간단한 연산을 구분해서 methods와 computed로 나눠서 구현함
### :zap: 아쉬운 점
- 아직 TDD의 개념이 확실하지 않아서 자료를 많이 찾아보았는데도 사람마다 기준이 달라서 명확한 답은 얻을 수 없었다. TC를 짤 때 최대한 내부 소스가 변경되도 TC는 변경되지 않는 구조로 짜려고 노력을 하였지만 데이터 값을 테스트 할 때는 setData()를 이용해서 임의의 데이터를 넣어주고 테스트를 진행하였다. 하지만 이 방법이 최선인지, 더 좋은 방법이 있는데 못찾는 것인지 확신이 없는 게 아쉬웠다.

## 3주차
### :bulb: 쇼핑몰 상품 목록 페이지 구현하기
<img src="https://user-images.githubusercontent.com/32696209/150643936-71676035-a8c2-4ab2-919f-2577f0289333.png" width="250" />

### :zap: 접근 방법
- 주어진 요구사항에 대한 상품 상세 정보 페이지에 대한 기획을 함
- 헤더, 상품아이템, 하단바 세가지로 컴포넌트 구성
- props를 이용해 컴포넌트 간 데이터를 주고받음
- lifecycle(mounted(), beforeUnmount())를 이용해서 스크롤 이벤트 구현
- flex를 사용해서 상품리스트를 2열로 표시
- 이미지가 없을 경우에 대한 예외처리 추가
- 2주차에서는 할인에 대한 내용을 true, false로 설정해서 할인율과 할인가를 보여지게 했는데 이번 주차에서는 판매가와 원가 둘 다 있으면 할인율을 계산한 뒤 보여지게 하고, 원가만 있으면 원가가 판매가로 보여지게 설정
### :zap: 아쉬운 점
- 2주차에서 할인가를 계산할 때 computed를 사용했었는데 이번 주차에서는 할인율을 계산할 때 원가와 판매가를 매개변수로 받아와 계산하기 위해 method를 사용했는데, 이게 적절한 방법인지 확신이 없는게 아쉽다.

## 4주차
### :bulb: 페이지 라우팅 및 API 연동
<img src="https://user-images.githubusercontent.com/32696209/152640281-25c24222-c5a6-4418-b9bb-a07c8a8d3a14.png" width="250" />

### :zap: 접근 방법
- vue-router를 사용해서 화면을 전환시킴
- API를 연동할 때 axios를 사용함
- API를 연동을 할 때 Repository pattern을 활용함
- 상세페이지와 내비게이션의 메뉴들을 API를 연동해서 화면을 전환시킴
- 라우터 테스트는 real router를 사용해서 테스트함
### :zap: 아쉬운 점
- api호출에 대한 테스트와 api를 연동 한 후 get이외의 기능들을 구현하지 못한 점이 아쉬움
